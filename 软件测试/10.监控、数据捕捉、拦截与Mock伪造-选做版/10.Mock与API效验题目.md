## 一、简答题

### 1. 监控与捕捉对比
- 简述【监控模式 `page.on` 】与【数据捕捉 `page.expect_response` 】的使用场景及代码区别？

  `page.on("response", callback)`：用于**持续监听**所有网络响应，适合做性能审计、错误监控等全局行为分析。

  `page.expect_response(url_or_predicate)`：用于**等待并捕获特定请求的响应**，常用于断言接口返回内容是否符合预期。

- 深度思考：为什么有了 UI 断言（如 `expect(locator).to_have_text("成功")`）还需要这两种手段？

  - UI 断言只能验证最终呈现结果，无法判断数据来源是否正确、接口是否异常或资源是否过大。
  - 网络层控制可验证：
    - 接口是否被正确调用；
    - 返回的数据结构和状态码是否合规；
    - 静态资源加载是否影响性能。

### 2. 拦截(`page.route`)
- `page.route()` 的调用位置为什么非常关键？如果放在 `page.goto()` 之后会发生什么？

  必须在 `page.goto()` **之前注册 route**，否则页面已发出请求，拦截失效。

  若放在之后，可能错过初始加载阶段的请求（如 HTML 中 script 或 img 标签触发的请求）。

- 给出拦截所有图片请求（.png/.jpg/.gif）的参数写法。

  ```python
  page.route(re.compile(r"\.(png|jpg|jpeg|gif)$"), lambda route: route.abort())
  ```

  

- 简述 `route.abort()`（丢弃请求）与 `route.fulfill()`（伪造响应）的应用场景。

  `abort()`：模拟网络失败、资源屏蔽（如广告拦截）。

  `fulfill()`：完全伪造响应体、状态码、Header，用于 Mock 测试。

### 3. Mock
- 给出 Mock 的 API 及应用场景。

  - 主要 API：`page.route(url, handler)`
  - 应用场景：
    - 第三方服务不可用时保证测试稳定性；
    - 模拟异常状态（如 500 错误、超时）；
    - 构造特定数据（如负余额、超长字段）验证前端鲁棒性。

- 如何利用 `route.continue_` 注入 Header（如 Token）或模拟网络延迟？

  ```python
  def handle(route):
      route.continue_(headers={**route.request.headers, "Authorization": "Bearer test-token"}, 
                      delay=500)  # 模拟 500ms 延迟
  page.route("/api/admin/info", handle)
  ```

  

- `route.continue_` 修改 Header 时，是合并还是覆盖？

  是**合并**而非覆盖。若需覆盖，应显式传入完整 headers 字典。

- 简述 `route.fetch()` 的作用，以及它如何实现“基于真实数据的局部篡改”？

  发起一个**真实的下游请求**（绕过当前拦截），获取原始响应后再修改部分字段。

  实现“半 Mock”：保留真实数据结构，仅篡改特定字段（如用户名长度），用于边界测试。

### 4. 工业级实践
- 在工业级测试中，为什么不建议 Mock 所有的接口？请列举至少两个“必须 Mock”的场景和两个“建议走真实接口”的场景。
  - **为什么不建议全量 Mock**？
    - 过度 Mock 会导致测试脱离真实环境，掩盖集成问题（如 API 字段变更、认证机制调整）。
    - 真实链路才能暴露网络延迟、跨域、CORS、证书等问题。
  - **必须 Mock 的场景**：
    1. 支付、短信、邮件等**不可逆或收费操作**；
    2. 第三方服务**无测试环境**或 SLA 不稳定。
  - **建议走真实接口的场景**：
    1. **核心业务流程验证**（如登录、下单）；
    2. **端到端集成测试**，需验证前后端契约一致性。
- `storage_state` 和 `route.continue_` 都可以实现免登陆，分别适用于什么场合？
  - `storage_state`：适用于**已通过真实登录流程**后保存 Cookie/LocalStorage，后续测试直接复用，更贴近用户真实状态。
  - `route.continue_` 注入 Token：适用于**无需真实登录**、仅需模拟权限的轻量级测试，或测试 Token 本身逻辑（如过期、格式错误）。

### 实战 1：性能与错误审计监控 (page.on)
**要求**：监听页面所有响应，实现以下自动化审计逻辑：

1. **静态资源检查**：若任何 `png` 或 `jpg` 资源的大小超过 1MB，在控制台输出警告 URL。
2. **错误监控**：若任何接口状态码 ≥ 400，记录该接口的 URL 和状态码内容。

```python
# test01.py
import re
from playwright.sync_api import sync_playwright

def test_audit_monitor():
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=False)
        page = browser.new_page()

        large_images = []
        errors = []

        def on_response(response):
            if response.status >= 400:
                errors.append(f"{response.url} -> {response.status}")
            if re.search(r"\.(png|jpg|jpeg)$", response.url):
                try:
                    # 尝试获取 body 大小（即使无 content-length）
                    body_size = len(response.body())
                    if body_size > 1_000_000:
                        large_images.append(f"{response.url} ({body_size} bytes)")
                except Exception as e:
                    # 跨域资源无法读取 body，跳过
                    pass

        page.on("response", on_response)
        page.goto("http://localhost:8000/10.Mock_API_Practice.html")
        page.get_by_role("button", name="触发审计测试").click()
        page.wait_for_timeout(5000)  # 延长等待时间

        print("❌ 错误接口:", errors)
        print("🖼️ 大图资源 (>1MB):", large_images)

        assert len(errors) > 0, "应捕获到 /api/v1/private-data 的 404 错误"
        # 如果你确定本地有大图，保留下面断言；否则注释
        # assert len(large_images) > 0, "应捕获到 >1MB 的图片"

        browser.close()

if __name__ == "__main__":
    test_audit_monitor()
```



### 实战 2：数据双重校验 - Mock 与精准断言 (page.route + expect_response)
**场景**：点击“刷新余额”按钮后，前端会访问 `/api/user/balance`。
**要求**：

1. **Mock 注入**：利用 `page.route` 拦截并返回 JSON 数据 `{"amount": -100.0, "currency": "CNY"}`。
2. **同步捕捉**：在点击按钮的同时，使用 `expect_response` 捕获该接口响应。
3. **复合断言**：
   - **UI 层**：验证页面显示“账户已欠费”的红色提示。
   - **数据层**：校验捕获到的 Response Body，确保 `amount` 等于 -100 且 `currency` 为 "CNY"。

```python
# test02_practice2_balance.py
from playwright.sync_api import sync_playwright
from playwright.sync_api import expect

def test_balance_mock_and_assert():
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=False)
        page = browser.new_page()

        def mock_balance(route):
            route.fulfill(json={"amount": -100.0, "currency": "CNY"})

        # 拦截生产环境余额接口
        page.route("https://api.production.com/api/user/balance", mock_balance)
        page.goto("http://localhost:8000/10.Mock_API_Practice.html")

        with page.expect_response("https://api.production.com/api/user/balance") as response_info:
            page.get_by_role("button", name="刷新余额").click()

        expect(page.locator("#balance-warning")).to_be_visible()
        # ✅ 修复：使用 to_contain_class 或完整类名
        expect(page.locator("#balance-card")).to_contain_class("danger-ui")

        data = response_info.value.json()
        assert data["amount"] == -100.0
        assert data["currency"] == "CNY"

        browser.close()

if __name__ == "__main__":
    test_balance_mock_and_assert()
```



### 实战 3：环境流量劫持与重定向 (page.route)
**场景**：前端代码硬编码指向生产环境 `https://api.production.com/v1/login`。
**任务**：编写拦截规则，实现以下目标：

1. 发往生产环境的请求自动重定向至测试环境 `https://api.test-env.com`。

2. 验证 Playwright 拦截层面的请求 URL 已被成功改写。（*注：真实浏览器发出重定向后可能会在控制台报跨域或 DNS 错误，忽略即可*）

   ```python
   # test03_practice3_redirect.py
   from playwright.sync_api import sync_playwright
   
   def test_redirect_production_to_test():
       with sync_playwright() as p:
           browser = p.chromium.launch(headless=False)
           page = browser.new_page()
   
           def redirect_route(route):
               original = route.request.url
               new_url = original.replace("api.production.com", "api.test-env.com")
               print(f"✅ 拦截成功！重写 URL:\n  原始: {original}\n  新URL: {new_url}")
               route.continue_(url=new_url)
   
           page.route("https://api.production.com/**/*", redirect_route)
           page.goto("http://localhost:8000/10.Mock_API_Practice.html")
   
           # 触发请求（来自 checkAuth 函数中的 fetch）
           page.evaluate("fetch('https://api.production.com/v1/login').catch(()=>{})")
           page.wait_for_timeout(1000)
   
           browser.close()
   
   if __name__ == "__main__":
       test_redirect_production_to_test()
   ```

   

### 实战 4：动态权限矩阵测试 (route.continue + fulfill)
**要求**：
1. **Header 注入**：拦截获取管理信息请求，动态注入 `{"Authorization": "Bearer {token}"}`。

2. **结果模拟**：
   - `admin` 角色：Mock 返回 `status=200` 并验证 UI 出现管理按钮。
   
   - `guest` 角色：Mock 返回 `status=403` 并验证 UI 提示权限不足。
   
     ```python
     # test04_practice4_auth.py
     import sys, re
     from playwright.sync_api import sync_playwright
     from playwright.sync_api import expect
     
     def test_auth_matrix(role: str):
         print(f"🧪 测试角色: {role}")
         with sync_playwright() as p:
             browser = p.chromium.launch(headless=False)
             page = browser.new_page()
     
             def handle_admin(route):
                 headers = {**route.request.headers, "Authorization": f"Bearer {role}_token"}
                 if role == "admin":
                     route.fulfill(status=200, json={"role": "admin"})
                 else:
                     # ✅ 修复：用 body 而不是 text
                     route.fulfill(status=403, body="Forbidden")
     
             page.route("**/api/admin/info", handle_admin)
             page.goto("http://localhost:8000/10.Mock_API_Practice.html")
             page.get_by_role("button", name=re.compile(r"获取管理信息")).click()
     
             if role == "admin":
                 expect(page.locator("#admin-content")).to_be_visible()
                 expect(page.locator("#auth-error")).not_to_be_visible()
             else:
                 expect(page.locator("#admin-content")).not_to_be_visible()
                 expect(page.locator("#auth-error")).to_be_visible()
     
             page.wait_for_timeout(3000)
             browser.close()
     
     if __name__ == "__main__":
         role = sys.argv[1] if len(sys.argv) > 1 else "admin"
         if role not in ["admin", "guest"]:
             print("Usage: python test04.py [admin|guest]")
             sys.exit(1)
         test_auth_matrix(role)
     ```
   
     

### 实战 5：基于真实数据的“半 Mock”篡改 (route.fetch)
**场景**：测试“超长用户名”导致的 UI 渲染排版问题（数据源：真实的公开 API `https://jsonplaceholder.typicode.com/users`）。
**要求**：

1. **数据中转**：利用 `route.fetch()` 向该公开 API 获取**真实**后端数据。

2. **局部篡改**：仅修改返回 JSON 数组中第一个用户的 `name` 字段为 100 个字符长度，其他字段保持原样。

3. **验证**：页面能正常渲染出真实数据和被修改过的超长内容。

   ```python
   # test05_practice5_semi_mock.py
   from playwright.sync_api import sync_playwright
   from playwright.sync_api import expect
   
   def test_semi_mock_long_username():
       with sync_playwright() as p:
           browser = p.chromium.launch(headless=False)
           page = browser.new_page()
   
           def handle_users(route):
               real_res = route.fetch()
               users = real_res.json()
               users[0]["name"] = "Mocked_User_" + "X" * 87  # 总长 100
               route.fulfill(
                   status=real_res.status,
                   headers=real_res.headers,
                   json=users
               )
   
           page.route("https://jsonplaceholder.typicode.com/users", handle_users)
           page.goto("http://localhost:8000/10.Mock_API_Practice.html")
           page.get_by_role("button", name="获取真实用户列表").click()
           page.wait_for_selector("#user-list li", timeout=10000)
   
           first_user = page.locator("#user-list li").first
           expect(first_user).to_contain_text("Mocked_User_XXXXXXXXXX")  # 验证篡改成功
   
           browser.close()
   
   if __name__ == "__main__":
       test_semi_mock_long_username()
   ```
